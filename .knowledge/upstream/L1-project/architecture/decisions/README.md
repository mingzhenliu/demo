# 架构决策记录 (ADR)

## 概述

本目录存放项目的架构决策记录 (Architecture Decision Records)。

## ADR 清单

| ADR 编号 | 标题 | 状态 | 日期 |
|----------|------|------|------|
| ADR-000 | ADR 模板 | 已采纳 | - |

## ADR 状态说明

| 状态 | 说明 |
|------|------|
| 提议中 | 正在讨论中的决策 |
| 已采纳 | 已通过并实施的决策 |
| 已弃用 | 被新决策取代 |
| 已取消 | 讨论后决定不采纳 |

## ADR 模板

创建新 ADR 时，请使用以下模板：

```markdown
# ADR-XXX: [决策标题]

## 状态

[提议中 | 已采纳 | 已弃用 | 已取消]

## 背景

[描述导致此决策的背景和问题]

## 决策

[描述所做的决策]

## 理由

[解释为什么做出这个决策，考虑了哪些替代方案]

## 影响

### 正面影响
- [列出正面影响]

### 负面影响
- [列出负面影响或需要注意的点]

## 相关

- [相关的 ADR 或文档链接]

---
决策日期: YYYY-MM-DD
决策人: @xxx
```

## 命名规范

- 文件名: `ADR-XXX-简短描述.md`
- 编号: 三位数字，从 001 开始递增
- 描述: 使用短横线分隔的小写单词

示例: `ADR-001-use-postgresql-as-primary-database.md`

## 维护说明

- 重大架构变更必须记录 ADR
- ADR 一旦采纳不应删除，只能标记为弃用
- 定期 Review 已采纳的 ADR 确保仍然适用
