## 流程文档模板

每个业务流程文档应包含以下内容：

1. **流程概述**: 流程目的和范围
2. **参与角色**: 涉及的用户角色和系统
3. **流程图**: 使用流程图展示步骤
4. **步骤详情**: 每个步骤的详细说明
5. **异常处理**: 异常情况及处理方式
6. **相关接口**: 涉及的 API 接口

## 示例流程文档

请参考以下模板创建流程文档：

```markdown
# [流程名称]

## 1. 流程概述
[描述流程的目的和业务价值]

## 2. 参与角色
- 角色 A: [职责说明]
- 角色 B: [职责说明]

## 3. 流程图
[使用 Mermaid 或 ASCII 图]

## 4. 步骤详情
### 步骤 1: [步骤名称]
- 触发条件:
- 执行动作:
- 输出结果:

## 5. 异常处理
| 异常场景 | 处理方式 |
|----------|----------|

## 6. 相关接口
- POST /api/xxx - [接口说明]
```

## 维护说明

- 新增业务流程需同步创建流程文档
- 流程变更需更新文档并通知相关团队
- 复杂流程建议配合时序图说明
